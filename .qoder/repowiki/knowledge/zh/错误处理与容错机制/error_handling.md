该代码库采用**轻量级、脚本化**的错误处理策略，主要依赖 Node.js 原生的 `try...catch` 机制和**结构化返回对象**。由于项目定位为“智能运营套件”（CLI/自动化脚本为主），未引入复杂的中间件或全局异常捕获框架，而是通过**兜底指引（Fallback Guides）**、**自动修复工具**和**环境自检**来保障系统的鲁棒性。

### 1. 核心处理模式
*   **结构化返回 (Result Object Pattern)**：在核心业务逻辑（如 `ota-operator.js`）中，函数不直接抛出异常，而是返回包含 `success`, `error`, `message` 及可选 `fallbackGuide` 的对象。调用方根据 `success` 字段决定后续流程。
*   **Try-Catch 边界防护**：在异步操作（如 Playwright 浏览器控制、文件 I/O、网络请求）的最外层使用 `try...catch`。捕获异常后，记录错误信息并返回失败状态，防止进程崩溃。
*   **兜底人工指引 (Fallback Strategy)**：当自动化操作（如 OTA 改价）失败时，系统不仅返回错误原因，还会生成一份详细的**人工操作指引**（`generateFallbackGuide`），指导用户手动完成任务，体现了“人机协同”的容错设计。

### 2. 关键实现细节
*   **浏览器自动化容错**：
    *   在 `ota-operator.js` 中，针对 DOM 选择器可能失效的情况，显式返回 `needsCalibration: true` 标记。
    *   操作前后自动截图存证（`takeScreenshot`），便于故障排查。
    *   登录态检测：在执行写操作前校验页面文本，若发现登录过期则提前终止并提示重新初始化。
*   **数据完整性与修复**：
    *   `fix-json.js` 提供了专门的 JSON 修复工具，能自动检测并修复尾逗号、BOM 标记、引号错误等常见语法问题，并在修复前自动备份（`.bak`）。
    *   `check-env.js` 执行 10 项环境自检，包括磁盘空间、依赖包、配置文件语法等，将错误预防前置。
*   **配置写入安全**：
    *   `config-writer.js` 采用“读取→合并→写入”模式，避免覆盖无关字段。
    *   提供严格的字段验证器（`isValidPhone`, `isValidAmount` 等），在写入前拦截非法数据，返回标准化的 `{ success: false, error: ... }` 结果。
*   **网络与通知容错**：
    *   `notifier.js` 在企业微信通知发送失败时，捕获网络错误并返回详细的状态码（`errcode`/`errmsg`）。
    *   支持 Webhook URL 缺失时的优雅降级（仅打印错误日志，不中断主流程）。

### 3. 开发者规范
*   **禁止静默失败**：所有 `catch` 块必须记录错误日志或返回明确的错误对象，严禁空 `catch`。
*   **统一错误出口**：CLI 入口函数（`main`）通常以 `.catch(err => { console.error(...); process.exit(1) })` 结尾，确保未捕获异常能友好退出。
*   **验证前置**：在执行耗时操作（如启动浏览器）前，先检查配置文件、登录态等前置条件，尽早失败（Fail-fast）。
*   **兜底思维**：对于涉及外部系统（OTA 平台）的操作，必须考虑自动化失败后的备选方案（如生成人工指引）。
