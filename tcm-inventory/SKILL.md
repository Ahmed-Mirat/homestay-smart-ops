# 中医馆进销存管理模块

## 功能概览

| 功能 | 触发示例 | 说明 |
|------|---------|------|
| 产品/服务目录 | "新增产品: 艾条 艾灸耗材 进货价8元/盒 售价20元/盒" | 增删改查产品信息 |
| 库存入库 | "入库 艾条×50盒 生产日期2026-06-15 有效期12个月" | 进货入库，自动记录效期 |
| 开单扣减 | 开单时自动关联库存扣减 | 与收银联动，无需手动 |
| 库存查询 | "查库存" / "艾条还剩多少" | 实时库存 + 低库存/临期标记 |
| 库存预警 | 库存低于阈值自动推送企微通知 | 阈值可自定义，默认10 |
| 效期管理 | "查效期" / "哪些快过期了" | 录入生产日期+有效期，提前30天预警 |

---

## 🤖 Agent 行为规则

### 触发词识别与处理流程

收到库存相关触发词后，Agent 按以下流程处理：

1. **识别意图** — 根据触发词判断操作类型：
   - 查询类："查库存" / "还剩多少" / "什么快没了" / "查效期" → 调用 `list` / `lowStock` / `expiring`
   - 入库类："入库" / "进货" / "补货" / "新增产品" → 调用 `add` / `stockIn`
   - 出库类："出库" / 开单关联扣减 → 调用 `deduct`
   - 预警类："库存预警" / "临期" → 调用 `check`

2. **参数提取** — 从用户自然语言中提取产品名/ID、数量、批次等参数

3. **执行操作** — 调用 `inventory.js` 对应命令

4. **结果展示** — 将返回值转为用户友好的自然语言

### 库存不足时的处理方式

当 `deduct` 返回 `success: false` 时：

1. 向用户展示警告信息（如 "⚠️ 艾条库存仅剩3盒，需扣减5盒，库存不足"）
2. **不会自动执行扣减**，询问用户："库存不足，是否仍要强制扣减？"
3. 用户确认后，可手动调整库存或选择跳过

### 预警通知自动触发条件

以下情况自动触发企业微信预警推送：
- 扣减后库存 ≤ 该产品的 `minStock` 阈值
- 通知通过 `notifier.js` 异步发送，不阻塞主流程
- 若 notifier 不可用，仅在对话中提示，不影响扣减操作

## 1. 产品/服务目录

### 新增产品

```
用户: 新增产品: 艾条 艾灸耗材 进货价8元/盒 售价20元/盒 库存50盒 生产日期2026-06-15 有效期12个月 最低库存10盒
```

Agent 调用 `inventory.js add` 写入 `_shared/data/inventory.json`，字段：

| 字段 | 必填 | 示例 | 说明 |
|------|:---:|------|------|
| name | ✅ | 艾条 | 产品名称 |
| category | ✅ | 艾灸耗材 | 分类：艾灸耗材/中药饮片/中成药/医疗器械/保健品/其他 |
| purchasePrice | ✅ | 8 | 进货单价(元) |
| sellPrice | ✅ | 20 | 售价(元) |
| stock | ✅ | 50 | 当前库存数量 |
| unit | ✅ | 盒 | 单位(盒/包/瓶/支/袋/个) |
| productionDate | 选填 | 2026-06-15 | 生产日期 |
| expiryMonths | 选填 | 12 | 有效期(月) |
| minStock | 选填 | 10 | 最低库存阈值 |

### 产品分类

默认可选分类：艾灸耗材 / 中药饮片 / 中成药 / 医疗器械 / 保健品 / 其他

### 查看目录

```
用户: 产品目录
用户: 看看艾灸类的有什么
```

### 修改产品

```
用户: 修改产品: 艾条 售价改为22元
用户: 删除产品: 过期艾条
```

## 2. 库存入库

```
用户: 入库 艾条×30盒 生产日期2026-07-01 有效期12个月
用户: 入库: 当归×20包(进货价15) 黄芪×15包(进货价12)
```

Agent 调用 `inventory.js stockIn` 更新库存。如果产品不存在，先引导录入产品信息。

---

## 3. 开单自动扣减

开单时（如"开单: 张阿姨 针灸调理"），如果诊疗项目关联了耗材（如艾灸→消耗艾条1盒），Agent 自动：

1. 创建消费单据（原有收银流程）
2. 调用 `inventory.js deduct` 扣减关联耗材库存
3. 如果库存不足，提示"⚠️ 艾条库存仅剩3盒，请及时补货"
4. 如果扣减后低于最低库存阈值，触发企微预警

关联关系在产品目录中配置：

```
用户: 设置: 艾灸调理消耗艾条1盒
```

## 5. 库存预警

### 低库存预警

当库存 ≤ 最低库存阈值时：
1. Agent 主动提醒："⚠️ 当归库存仅剩3包(阈值5包)，建议补货"
2. 如果已配置企业微信通知 → 推送告警

```
📱 企微通知:
【库存预警】当归库存仅剩3包(最低阈值5包)
建议进货量: 20包
上次进货: 6月15日(已45天)
```

### 自定义阈值

```
用户: 设置: 当归最低库存5包
用户: 所有艾灸耗材最低库存统一设10
```

## 6. 效期管理

### 录入效期

入库时自动记录：
- `productionDate`：生产日期
- `expiryDate`：自动计算 = 生产日期 + 有效期(月)

### 效期查询

```
用户: 查效期
→ 全部产品的效期状态

用户: 哪些快过期了
→ 临期清单(30天内):
  艾条(批号20250601) 有效期至2026-06-01 ⚠️ 已过期!
  当归(批号20251115) 有效期至2026-11-14 ⚠️ 30天后到期
```

### 过期预警

Agent 每天首次对话时自动扫描：
- 已过期 → "⚠️ 以下产品已过期，请处理: 艾条(批号20250601)"
- 30天内到期 → "📅 以下产品30天内到期: 当归(批号20251115)"

## 7. 返回值格式说明

### deduct（出库扣减）

**库存充足 — 正常扣减：**
```json
{ "success": true, "product": { "id": "inv-001", "name": "艾条", "stock": 46, ... } }
```

**库存充足但触发低库存预警：**
```json
{
  "success": true,
  "product": { "id": "inv-001", "name": "艾条", "stock": 8, ... },
  "warning": "⚠️ 艾条库存8盒，低于阈值10盒，建议补货"
}
```

**库存不足 — 拒绝扣减：**
```json
{
  "success": false,
  "product": { "id": "inv-001", "name": "艾条", "stock": 3, ... },
  "warning": "⚠️ 艾条库存不足（需5盒，仅剩3盒），请先补货"
}
```

### add（新增产品）

**成功：** 返回完整的产品对象（含自动生成的 `id` 和 `expiryDate`）

**失败抛出异常：**
- JSON 格式错误：`"产品JSON格式错误: Unexpected token ..."`
- 缺少必填字段：`"产品名称(name)为必填项"` / `"产品分类(category)为必填项"`

### findProduct（产品查找）

- 精确匹配 ID → 返回产品对象
- 名称唯一匹配 → 返回产品对象
- 同名产品多于1个 → 抛出异常：`"存在2个同名产品'当归'，请使用产品ID进行操作"`
- 未找到 → 返回 `null`

## 8. 集成说明（notifier.js）

### 低库存自动推送

进销存模块已集成 `_shared/scripts/notifier.js`，当扣减后库存触发预警阈值时，自动向企业微信群推送告警：

```
📱 企微通知:
【库存预警】艾条库存8盒，低于安全阈值10盒，请及时补货
```

### 通知降级机制

```javascript
// notifier.js 不可用时优雅降级
let notifyAlert = null;
try {
  const notifier = require('../../_shared/scripts/notifier');
  notifyAlert = notifier.notifyAlert || notifier.default?.notifyAlert || null;
} catch (e) {
  // 静默降级，不输出错误信息
}
```

- `notifier.js` 加载失败（如未安装依赖）→ 主流程正常执行，仅跳过通知
- 通知发送失败（如 webhook 无效）→ 控制台输出警告，不抛异常
- 通知为异步发送，不阻塞库存扣减操作

## 9. 错误处理示例

以下为常见错误场景及 Agent 应如何向用户展示：

### 产品不存在

```
用户: 出库 板蓝根×10包
Agent: ❌ 未找到产品"板蓝根"，请确认产品名称是否正确，或使用"产品目录"查看现有产品。
```

### 库存不足

```
用户: 出库 艾条×50盒
Agent: ⚠️ 艾条库存不足（需50盒，仅剩8盒），未执行扣减。
      是否需要：1) 先补货再扣减  2) 按现有库存全部出库(8盒)？
```

### 同名产品冲突

```
用户: 查库存 当归
Agent: ⚠️ 系统中存在2个同名产品"当归"：
      - inv-003: 当归（中药饮片）库存200克
      - inv-007: 当归（中成药）库存15盒
      请指定产品ID进行操作，如"查库存 inv-003"
```

### JSON 格式错误

```
用户: 新增产品 {name: 艾条}
Agent: ❌ 产品信息格式有误，请按以下格式提供：
      新增产品: 艾条 艾灸耗材 进货价8元/盒 售价20元/盒 库存50盒
      我会自动转换为正确格式录入。
```
