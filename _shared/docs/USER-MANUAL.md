# 民宿智能运营助手 — 内部使用手册（Agent 参考）

> 此文档为 Agent 在商户问"有什么功能/能干什么/怎么用"时的内部参考，不直接展示原文给商户。
> Agent 应阅读相关章节后用对话化口语回答，并根据商户的具体问题精准定位到对应功能。

---

## 套件定位

民宿智能运营 AI Skill 套件，安装后通过对话使用。
**8 项核心功能无需商家后台账号**（竞品采集仅需消费者端账号），5 项商家后台操作功能可后续激活。

---

## 安装路径（3 步完成）

### 第 1 步：放入工作目录
将 ZIP 拖入 QoderWork 技能导入框，识别到 SKILL.md 即生效。

### 第 2 步：安装依赖
QoderWork 终端中执行 `cd _shared && npm install`，约 1 分钟。
首次对话时 SETUP-WIZARD 第 0 步会自动检测并指导执行。

### 第 3 步：完成设置向导
对话中说 **"开始设置"**，按 5 步引导回答（约 10 分钟）：
1. 民宿名称和地址（1 分钟）
2. 房型详情（3 分钟）
3. 入住规则（2 分钟）
4. 周边交通（2 分钟）
5. 团队配置 + 通知（2 分钟）

完成后 8 项核心功能立即可用。

## 立即可用功能（8 项，无需商家后台账号）

### 💬 智能客服
| 商户/客人说 | 助手做什么 |
|------|-----------|
| "WiFi密码是什么" | 基于知识库精准回答 |
| "可以带宠物吗" | 读取入住规则回答 |
| "怎么到你们那" | 输出交通指引 |
| "客人说房间太差了要投诉" | 触发愤怒处理流程：共情+降温+转人工 |
| "帮我回一条差评，客人说卫生差" | 生成差评回复模板 |

### 🧹 保洁排班
| 商户说 | 助手做什么 |
|------|-----------|
| "小王打扫201、202，小李打扫301" | 记录排班并自动创建保洁任务 |
| "帮我排班" | 读取退房房间+员工列表，建议均衡分配 |
| "本周排班统计" | 输出每人完成量 |

### 📋 任务看板
| 商户说 | 助手做什么 |
|------|-----------|
| "帮我创建一个维修任务，502房马桶" | 创建维修任务 |
| "301完成了" | 标记 301 任务完成 |
| "保洁都做完了" | 批量完成所有保洁任务 |
| "打开任务看板" | 生成三列看板（待办/进行中/已完成）的 HTML 文件并提示双击打开 |

### 📱 通知推送（企业微信群）
配置 webhook 后自动推送：
- 任务派发："🧹 301房 退房清洁 @小王"
- 任务完成："✅ 301房保洁已完成"
- 日终摘要：今日任务统计+明日待办
- 告警：任务超时提醒

配置方法：建群→添加群机器人→复制 webhook 链接→发给助手。
**没有企业微信也不影响核心功能**，所有通知在 QoderWork 对话和工作台面板中都能看到。

### 🌙 日终流程
| 商户说 | 助手做什么 |
|------|-----------|
| "日终" | 统计今日完成情况+推送微信群+生成明日待办 |

也可设置每天 22:00 自动执行（cron-scheduler）。

### 📊 工作台面板
| 商户说 | 助手做什么 |
|------|-----------|
| "打开工作台" | 生成 KPI+待办+告警的面板 HTML，提示双击打开 |

### 📝 手动订单管理
| 商户说 | 助手做什么 |
|------|-----------|
| "携程来了一个订单，张先生，山景大床房，6月5日到7日" | 自动写入 orders.json |
| "今天有哪些订单" | 展示今日订单 |
| "明天有入住吗" | 查询次日入住 |

口述录入的订单与 OTA 自动采集的格式完全相同，后续接入平台后无缝切换。

### 🔍 竞品价格采集（仅需消费者端账号）
| 商户说 | 助手做什么 |
|------|-----------|
| "采集竞品数据" | 打开携程/美团/飞猪/去哪儿/同程竞品详情页，自动提取房型价格 |
| "看看周边民宿卖多少钱" | 执行五平台采集并输出对比报告 |
| "配置竞品列表" | 引导商户填写竞品名称和各平台详情页链接 |

**仅需消费者端普通账号**（携程/美团/飞猪个人账号），不需要商家后台。
操作：先 `node scraper.js init` 登录一次，之后说"采集竞品数据"自动执行。
登录过期时助手会自动提示重新登录。

## 待激活功能（需商家后台账号）

| 功能 | 激活后能做什么 | 需要什么 |
|------|----------------|---------|
| 自动改价 | 确认后系统帮您直接在后台改价 | 携程/美团/飞猪商家后台 |
| 全平台关房/开房 | 一键在所有平台关闭或开放房间 | 各平台商家后台 |
| 超卖预防 | 新订单后自动关闭其他平台同期售卖 | 各平台商家后台 |
| 自动数据采集 | 订单/房态/营收数据自动读取 | 各平台商家后台 |
| 活动同步 | 促销活动全平台一键同步 | 各平台商家后台 |

**不提供商家后台账号的变通方式**：
- 改价/关房 → 助手输出人工操作指引，商户按步骤在后台操作
- 数据 → 商户口述营收和入住数据，助手照样生成报表
- 竞品价格 → 用消费者端账号自动采集（已可用）或商户口述

详细激活步骤见 `_shared/setup/OTA-ACTIVATION-GUIDE.md`。

## Agent 回答策略

商户问开放性问题时，助手不应直接列出全部功能，而是根据商户的角色和情境推荐最相关的 2-3 项：

- 商户首次使用 → 引导走 SETUP-WIZARD
- 商户问"有什么功能" → 简短介绍立即可用 8 项 + 一句"还有 5 项商家后台操作功能可后续激活"
- 商户问"怎么用 X" → 直接告诉对应触发词
- 商户报问题 → 触发故障自救（check-env.js）
