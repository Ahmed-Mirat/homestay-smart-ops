# 分级房租催缴

## 触发方式

| 触发词 | 行为 |
|--------|------|
| "催缴房租" / "房租催收" | 执行催缴检查，对符合条件的租客生成话术并推送 |
| "查看催缴规则" | 显示当前配置的催缴规则 |
| "修改催缴规则" | 对话式修改催缴参数 |
| "添加租客" | 录入新租客信息 |
| "标记已缴 [租客名]" | 手动标记租客已缴费 |
| "催缴统计" | 显示本月催缴状态汇总 |

## 核心逻辑

### 三档催收

| 档位 | 触发条件 | 话术基调 | 示例 |
|------|---------|---------|------|
| 提前提醒 | 距交租日 ≤ 3 天 | 友好提醒 | "XX您好，您的房租将于X月X日到期，提醒您提前安排～" |
| 到期催收 | 交租日当天未缴 | 正式催收 | "XX您好，您的房租今天到期，请尽快完成缴费，如有困难可与我沟通" |
| 逾期通知 | 逾期 1/3/7 天 | 递进式严肃 | "XX您好，您的房租已逾期X天，请于48小时内完成缴纳，以免影响正常居住" |

### 租客标签

| 标签 | 颜色 | 行为 |
|------|------|------|
| 优质 | 🟢 | 提前提醒话术更柔和；逾期宽限可延长 |
| 普通 | 🟡 | 标准话术 |
| 关注 | 🔴 | 提前提醒提前至5天；逾期3天即升级话术 |

### 推送方式

复用 `_shared/scripts/notifier.js` 推送到企业微信群。

## 数据模型

租客数据存储于 `_shared/data/tenants.json`：

```json
{
  "tenants": [
    {
      "id": "T001",
      "name": "张三",
      "room": "301",
      "phone": "138xxxx",
      "rent_day": 10,
      "amount": 2800,
      "tag": "优质",
      "last_paid": "2026-05-10",
      "status": "已缴"
    }
  ],
  "rules": {
    "early_days": 3,
    "overdue_levels": [1, 3, 7],
    "auto_push": true,
    "push_time": "09:00"
  }
}
```

## 操作流程

### 1. 初始化租客数据
商户通过对话录入或批量导入。Agent 将数据写入 `_shared/data/tenants.json`。

### 2. 执行催缴检查
Agent 读取 `tenants.json`，计算每个租客距交租日的天数，匹配对应档位，生成话术列表。

### 3. 推送
Agent 逐条调用 `notifier.js` 推送到微信群。推送前需商户确认。

## 定时任务

复用 `homestay-workflow/scripts/cron-scheduler.js`：
- 每天 09:00 自动检查催缴状态
- 有需催缴的租客时自动生成话术并推送

## 手动操作

- 商户说"催缴房租" → 立即执行检查
- 商户说"标记已缴 张三" → 更新该租客 `last_paid` 和 `status`
