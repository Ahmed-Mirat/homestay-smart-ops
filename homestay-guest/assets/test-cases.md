# 民宿智能客服 — 快速测试用例

> 用于验证 homestay-guest Skill 的核心能力是否正常工作。

---

## 测试用例1：住前咨询 — 房型推荐

**输入**：
你好，我们一家四口想订周末的房间，有什么推荐吗？两个大人两个小孩，大的8岁小的3岁。

**期望输出特征**：
- [x] 推荐适合家庭的房型（如亲子房/家庭房）
- [x] 说明推荐理由（空间、床型、儿童设施）
- [x] 报价标注为周末价
- [x] 来源标注 `[房源信息]`
- [x] 情绪等级：平静

---

## 测试用例2：入住指引生成

**输入**：
我订了明天晚上的山景大床房，能发一下入住指引吗？

**期望输出特征**：
- [x] 包含地址、入住时间、入住方式
- [x] 包含交通指引（至少2条路线）
- [x] 包含WiFi信息
- [x] 包含联系方式
- [x] 来源标注 `[入住规则]` `[周边信息]`

## 测试用例3：失物问询

- [x] 索要联系方式以便后续反馈
- [x] 来源标注 `[服务政策]`

## 测试用例4：轻微不满处理

## 测试用例5：严重投诉处理

**输入**：
太过分了！房间跟照片完全不一样，又小又脏！浴室还有头发！我要退款！这种条件也敢收这个价？马上给我处理！

**期望输出特征**：
- [x] 立即共情 + 降温
- [x] 不承诺退款（转交人工）
- [x] 不编造解决方案
- [x] 情绪等级：愤怒
- [x] 动作建议：🔴 紧急升级 - 需立即人工介入
- [x] 禁止自动处理退款

## 测试用例8：多轮对话上下文

**输入1**：你们有亲子房吗？
**输入2**：那个房间有浴缸吗？
**输入3**：好的，那我要订一间，怎么操作？

**期望输出特征**：
- [x] 第2轮能理解"那个房间"指亲子房
- [x] 第3轮能引导预订流程（联系前台/平台链接等）
- [x] 上下文连贯不断裂
